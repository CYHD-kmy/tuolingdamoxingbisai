# 「智投未来」A股日内投资智能体 — 产品设计书

## 一、项目概述

### 1.1 赛道目标

构建面向 A 股日内投资决策的智能体，每日输出可解释的投资建议，初始虚拟资金 50 万元，以赛期结束可用资金总额排名。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **复用优先** | 最大化复用 `TradingAgents-CN`、`ai-hedge-fund`、`daily_stock_analysis` 的成熟模块 |
| **可解释性** | 每笔决策附带完整推理链，从数据→信号→决策全链路可审计 |
| **稳健性** | 多层数据源降级、异常熔断、空仓兜底，极端行情下不崩溃 |
| **模块化** | 数据层 / 分析层 / 决策层 / 输出层 松耦合，各层可独立替换 |

## 三、核心模块设计

### 3.1 数据层 — UnifiedDataInterface

**复用来源**: `TradingAgents-CN` 的 `dataflows/interface.py` + `daily_stock_analysis` 的 `DataFetcherManager`

**设计要点**:

```python
class UnifiedDataInterface:
    """统一数据接口，多源降级"""

    # 数据源优先级 (从 daily_stock_analysis 复用)
    SOURCE_PRIORITY = [
        ("akshare", AKShareFetcher),     # 免费、无需Token、覆盖广 → 首选
        ("tushare", TushareFetcher),      # 需Token、质量高 → Token可用时升级
        ("baostock", BaoStockFetcher),    # 免费兜底
    ]

    def get_market_snapshot(self, codes: list[str]) -> MarketSnapshot:
        """获取全市场快照: 涨跌幅、量比、换手率、成交额"""

    def get_stock_daily(self, code: str, days: int = 60) -> DailyData:
        """获取个股日线: OHLCV + 均线 + MACD + KDJ + RSI"""

    def get_fund_flow(self, code: str, days: int = 5) -> FundFlow:
        """获取资金流向: 主力/超大单/大单/中单/小单 净流入"""

    def get_announcements(self, code: str, days: int = 7) -> list[Announcement]:
        """获取近期公告: 业绩预告、重大合同、增减持"""

    def get_news_sentiment(self, keywords: list[str]) -> NewsSentiment:
        """获取财经新闻情感: 正面/负面/中性 + 热度评分"""
```

**降级策略** (从 daily_stock_analysis 复用):
- AKShare 不可用 → 尝试 Tushare → 尝试 BaoStock → 返回缓存数据 → 标记数据质量等级
- 单只股票数据缺失不阻断整体流程

### 3.2 分析层 — Multi-Agent Analysis Pipeline

#### 阶段一: 海选筛选 (Morning Screening)

**复用来源**: `daily_stock_analysis` 的 `StockTrendAnalyzer`

**目标**: 从 5000+ A 股快速筛选 Top 20 候选，大幅降低 LLM 调用成本

**多因子打分模型**:

| 因子类别 | 因子名称 | 权重 | 说明 |
|----------|----------|------|------|
| 趋势 | 均线多头排列 | 15% | MA5>MA10>MA20>MA60 |
| 动量 | 5日涨幅 | 10% | 正向动量 |
| 量价 | 放量上涨 | 15% | 量比>1.5 且涨幅>2% |
| 资金 | 主力净流入 | 20% | 大单净流入占比 |
| 情绪 | 新闻正面率 | 10% | 近期新闻正面比例 |
| 质量 | ROE/毛利率 | 5% | 基本面过滤 |
| 风险 | 波动率适中 | 10% | 排除异常波动股 |
| 流动性 | 日均成交额 | 15% | >5000万，确保可交易 |

**筛选流程**:
```
5000+ 全市场
  → 剔除 ST/*ST/停牌/新股(上市<60天)
  → 剔除 日均成交额 < 5000万 (流动性过滤)
  → 多因子打分
  → Top 20 进入深度分析
```

#### 阶段二: 四维分析师并行分析

**复用来源**: `TradingAgents-CN` 的 `agents/analysts/` + `ai-hedge-fund` 并行 Agent 模式

**LLM 分层策略** (从 TradingAgents-CN 复用):

| 层级 | 模型 | 用途 | 调用频率 |
|------|------|------|----------|
| **quick** | DeepSeek-V3.2 / GPT-4o-mini | 分析师、研究员 | 高 (每只股票x4) |
| **deep** | DeepSeek-V4-Pro / Claude Opus | 研究主管、风控、组合主管 | 低 (全市场x1) |

四个分析师**并行运行** (从 ai-hedge-fund 复用)，各自使用 quick LLM:

```
                    ┌─ 技术面分析师 (Technical Analyst) ─┐
                    │  · 均线形态、MACD金叉死叉           │
                    │  · 布林带位置、RSI超买超卖          │
                    │  · 筹码分布、支撑压力位             │
候选池 (20只) ──────┼─ 基本面分析师 (Fundamentals Analyst) ─┼─→ 四份报告
                    │  · PE/PB分位数、ROE趋势             │
                    │  · 营收增速、利润质量               │
                    │  · 近期公告影响评估                 │
                    ├─ 资金面分析师 (Fund Flow Analyst) ───┤
                    │  · 主力资金动向、北向资金            │
                    │  · 融资融券变化                     │
                    │  · 大宗交易/龙虎榜                  │
                    └─ 消息面分析师 (News Sentiment Analyst)┘
                       · 财经新闻情感分析
                       · 行业/概念热点匹配
                       · 社交媒体热度
```

每个分析师遵循 **Tool-Calling 循环** (从 TradingAgents-CN 复用):
```
分析师LLM → 自主选择工具调用 → 获取数据 → 继续分析 → ... → 生成报告
```
最多 3 轮工具调用，防止无限循环。

#### 阶段三: 辩论式对抗

**复用来源**: `TradingAgents-CN` 的 Bull/Bear Researcher 辩论模式

```
多头研究员 (Bull Researcher):
  "基于技术面金叉+资金持续流入+业绩预增，
   建议买入 XXX，目标价 XX，止损 XX"
                    ↓
空头研究员 (Bear Researcher):        ← 基于多头论点逐条反驳
  "技术面虽有金叉但量能不足，
   资金流入可能为短期炒作，
   业绩预增已被市场price in..."
                    ↓
多头研究员 (Bull Researcher):        ← 回击空头论点
  ...
                    ↓
研究主管 (Research Manager, deep LLM):
  综合双方论点 → 给出最终研报
  {股票代码, 建议方向, 目标价, 置信度, 风险等级, 核心理由}
```

每只候选股票经过 1-3 轮辩论（可配置），研究主管用 deep LLM 做出最终研判。

### 3.3 决策层 — Risk-Adjusted Portfolio Construction

**复用来源**: `ai-hedge-fund` 的 `risk_manager.py` + `portfolio_manager.py`

#### 风控主管 (Risk Manager)

```python
class RiskManager:
    """风控规则 (确定性计算，不消耗 LLM Token)"""

    # 基本规则
    MAX_SINGLE_POSITION = 0.20    # 单票 ≤ 20% 总资金 (10万)
    MAX_INDUSTRY_EXPOSURE = 0.40  # 同行业 ≤ 40%
    MAX_DAILY_TURNOVER = 0.50     # 日换手率 ≤ 50% (避免过度交易)
    MAX_DRAWDOWN_DAILY = 0.05     # 日内回撤 5% 熔断
    MIN_CASH_RESERVE = 0.10       # 保留 ≥ 10% 现金

    def compute_position_limit(
        self,
        ticker: str,
        volatility: float,
        correlations: dict[str, float],
        current_portfolio: Portfolio
    ) -> PositionLimit:
        """
        波动率调整仓位上限 (从 ai-hedge-fund 复用)
        - 低波动 (σ < 2%): 1.25x 基准 → 最大 25%
        - 中波动 (2% ≤ σ < 4%): 1.00x 基准 → 最大 20%
        - 高波动 (σ ≥ 4%): 0.50x 基准 → 最大 10%

        + 相关性惩罚: 与已持仓股票高相关 (ρ > 0.7) → ×0.7
        """
```

### 3.4 输出层

#### 机器解析 JSON (赛道要求格式)

```json
[
  {
    "symbol": "600519",
    "symbol_name": "贵州茅台",
    "volume": 200
  },
  {
    "symbol": "000858",
    "symbol_name": "五粮液",

  }

```

**约束校验** (输出前自动检查):
- `volume` 必须是 100 的整数倍 → 自动向下取整
- 总买入金额 ≤ 可用现金 → 按优先级截断
- 标的必须可交易 → 交叉验证停牌列表
- 单票不超风控上限 → 自动裁剪

#### 人类可读日报

每交易日收盘后生成 Markdown 报告，包含:
1. **今日操作摘要**: 买入/卖出/持仓变动

3. **持仓快照**: 当前持仓 + 浮动盈亏
4. **明日关注**: 明日需重点关注的候选股

## 五、运行流程

### 5.1 交易日时间线

```
08:30  系统唤醒，检查数据源可用性
09:00  阶段一: 全市场海选 → Top 20 候选池
09:15  阶段二: 四维分析师并行分析 (每只股票 ~30s)
10:00  阶段三: 辩论式对抗 + 研究主管研判
10:30  阶段四: 风控计算 + 组合构建
10:45  输出竞JSON格式决策 → 提交至擂台平台
11:00  盘中监控启动 (可选: 熔断/止盈止损信号)
14:30  最终决策确认并输出
15:00  收盘后: 生成日报 + 更新记忆库 + 保存推理轨迹
```

### 5.2 异常处理

| 场景 | 处理策略 |
|------|----------|
| 数据源全部不可用 | 输出空数组 `[]`，标记"数据不可用" |
| LLM 调用超时 | 降级为确定性规则策略 (纯技术指标) |
| 单只股票分析失败 | 跳过该股票，继续分析其他候选 |
| 组合构建失败 | 输出空数组 `[]`，保留现有持仓 |
| 当日无符合条件标的 | 输出空数组 `[]` |

## 六、策略可解释性

### 6.1 推理链路可视化

每笔决策可从最终输出回溯到原始数据:

```
最终决策: 买入贵州茅台 200 股
  ↑
组合主管: "综合各维度看多信号，风控允许最大500股，建议保守配置200股"
  ↑
研究主管: "多头论点(业绩确定性+资金流入)强于空头(估值偏高+短期超买)，建议买入"
  ↑
├─ 多头研究员: "Q1业绩超预期20%，北向资金连续5日净流入，60日均线支撑有效"
├─ 空头研究员: "PE处于历史70%分位，近期涨幅过大RSI接近超买，短期有回调风险"
├─ 技术面: 均线多头排列，MACD金叉，成交量放大 → bullish(0.75)
├─ 基本面: ROE 28%，营收增速 18%，PE 32 (合理偏高) → bullish(0.82)
├─ 资金面: 主力净流入 2.3亿，融资余额上升 → bullish(0.68)
└─ 消息面: 研报覆盖增加，行业政策利好 → bullish(0.70)
```

## 九、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| AKShare 接口不稳定 | 数据缺失 | 三层降级 + 缓存兜底 + 标记数据质量 |
| LLM 调用成本过高 | 预算超支 | 海选筛选减少 deep LLM 调用 + quick/deep 分层 |
| 市场极端波动 | 大幅回撤 | 日内5%熔断 + 单票仓位上限 + 保留现金 |
| 过拟合历史数据 | 实盘表现差 | 辩论机制引入对抗观点 + 多家分析师交叉验证 |
| 擂台平台对接问题 | 无法提交 | 输出格式严格校验 + 模拟接口预测试 |
