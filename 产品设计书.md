# 「智投未来」A股日内投资智能体 — 产品设计书

## 一、项目概述

### 1.1 赛道目标

构建面向 A 股日内投资决策的智能体，每日输出可解释的投资建议，初始虚拟资金 50 万元，以赛期结束可用资金总额排名。

### 1.2 比赛规则

- **每日输出**: 在评测周期内，智能体按指定时间输出当日买入建议
- **资金结算**: 初始 50 万，按累计总资产排名。买入价 = 前一交易日收盘价，卖出价 = 当日收盘价，日终自动卖出不留隔夜持仓
- **策略可解释**: 逻辑清晰、假设明确、结果可审计与复现
- **输出格式**: 纯 JSON 数组 `[{symbol, symbol_name, volume}]`，空仓时返回 `[]`

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **博采众长** | 借鉴 `TradingAgents-CN`(多Agent辩论/LLM适配)、`ai-hedge-fund`(风控思路)、`daily_stock_analysis`(A股因子筛选) 的成熟模式，自研 A 股专属三层仓位框架 |
| **可解释性** | 每笔决策附带完整推理链，数据→信号→决策全链路可审计 |
| **稳健性** | 多层数据源降级、异常熔断、空仓兜底，极端行情下不崩溃 |
| **模块化** | 数据层 / 分析层 / 决策层 / 输出层 松耦合，各层可独立替换 |

## 三、核心模块设计

### 3.1 数据层 — UnifiedDataInterface

**设计要点**:

- **多源降级**: Tushare (主力) → BaoStock (备用) → AKShare (兜底)，按优先级自动降级
- **缓存**: TTL 缓存 + 磁盘持久化，减少重复请求
- **数据质量标记**: 每条数据标记 live/cached/fallback/stale

**核心数据接口**:

```python
class UnifiedDataInterface:
    """统一数据接口，多源降级 + 缓存 + 数据质量标记"""

    # ── 核心行情 ──
    def get_market_snapshot(self) -> list[MarketSnapshot]
    def get_daily_data(self, code: str, days: int = 60) -> list[StockDaily]
    def get_realtime_quote(self, code: str) -> Optional[RealtimeQuote]

    # ── 资金面 ──
    def get_fund_flow(self, code: str, days: int = 5) -> list[FundFlow]
    def get_northbound_flow(self, days: int = 5) -> list[NorthboundFlow]
    def get_northbound_stock(self, code: str, days: int = 10) -> list[dict]

    # ── 基本面 ──
    def get_financial_indicators(self, code: str) -> list[FinancialIndicator]
    def get_announcements(self, code: str, days: int = 7) -> list[dict]
    def get_unlock_shares(self, days_ahead: int = 30) -> list[UnlockShares]

    # ── 消息面 ──
    def get_news(self, keyword: str, days: int = 3) -> list[dict]

    # ── 批量接口 ──
    def batch_daily_data(self, codes, days=30, max_workers=8) -> dict
    def batch_fund_flows(self, codes, days=5, max_workers=8) -> dict
```

### 3.2 分析层 — Multi-Agent Analysis Pipeline

#### 阶段一: 海选筛选

**目标**: 从 5000+ A 股快速筛选 Top 20 候选，大幅降低 LLM 调用成本

**10因子打分模型**:

| 因子类别 | 因子名称 | 权重 | 说明 |
|----------|----------|------|------|
| 趋势 | 均线多头排列 | 12% | MA5>MA10>MA20>MA60 |
| 动量 | 5日涨幅 | 10% | 正向动量 |
| 量价 | 放量上涨 | 12% | 量比+涨幅连续评分 |
| 资金 | 主力净流入 | 15% | 大单净流入占比 |
| 北向 | 外资持仓变化 | 10% | 北向持股占比趋势 |
| 情绪 | 交易活跃度 | 8% | 换手率+连续阳线 |
| 质量 | PE+ROE+营收增速 | 10% | 多维度基本面过滤 |
| 风险 | 波动率适中 | 8% | 排除异常波动股 |
| 流动性 | 日均成交额 | 10% | >5000万，确保可交易 |
| 筹码 | 股东人数变化 | 5% | 筹码集中度指标 |

**筛选流程**: `5000+ → 剔除 ST/停牌/新股(<60天) → 剔除 日均成交额<5000万 → 10因子打分 → Top 20`

#### 阶段二: 四维分析师并行分析

四个分析师**并行运行**，各自使用 quick LLM:

| 分析师 | 分析维度 | 核心指标 |
|--------|----------|----------|
| 技术面 | 趋势形态、超买超卖 | 均线排列、MACD、RSI、布林带 |
| 基本面 | 估值水平、盈利质量 | PE/PB分位数、ROE趋势、营收增速 |
| 资金面 | 主力动向、北向资金 | 大单净流入、融资融券、龙虎榜 |
| 消息面 | 新闻情感、热点匹配 | 财经新闻分析、行业政策 |

每个分析师支持最多 3 轮 Tool-Calling 自主调用数据，防止无限循环。

#### 阶段三: 辩论式对抗

```
多头研究员: "基于技术面金叉+资金持续流入，建议买入..."
        ↓
空头研究员: 逐条反驳多头论点
        ↓
多头研究员: 回击空头论点 (max 3 轮)
        ↓
研究主管 (deep LLM): 综合双方论点 → 最终研报
  {direction, confidence, target_price, risk_level, core_reasoning}
```

### 3.3 决策层 — 风控约束 + 组合构建

#### 风控主管 (Risk Manager)

确定性规则计算，不消耗 LLM Token：

**三层仓位框架**:

| 层级 | 单票上限 | 说明 |
|------|----------|------|
| 核心仓 (Core) | ≤ 40% | 高置信度(≥0.75)标的 |
| 卫星仓 (Satellite) | ≤ 14% | 其他合格候选 |
| 现金备用 (Cash) | ≥ 25% | 始终保留现金缓冲 |

**风控调整因子**:
1. 分层基础仓位 (核心 40% / 卫星 14%)
2. 波动率调整 (低波×1.25 / 中波×1.0 / 高波×0.5)
3. 置信度调整 (0.5 + confidence × 0.5)
4. 风险等级调整 (low×1.0 / medium×0.8 / high×0.5)
5. 行业集中度检查 (同行业 ≤ 40%)
6. 相关性惩罚 (ρ > 0.7 → ×0.7)
7. 日换手率警告
8. 限售解禁减仓 (>5% ×0.3 / >2% ×0.6)

#### 投资组合主管 (Portfolio Manager)

使用 deep LLM 综合所有信号做出最终买入决策，输出比赛标准 JSON 格式：
```json
[{"symbol": "600519", "symbol_name": "贵州茅台", "volume": 200}]
```

### 3.4 输出层

#### 赛道 JSON（stdout）

直接输出到标准输出，无额外文本，符合比赛"直接只返回JSON"的要求。

```json
[{"symbol": "600519", "symbol_name": "贵州茅台", "volume": 200}]
```

约束校验（输出前自动执行）:
- `volume` 必须是 100 的整数倍 → 自动向下取整
- 总买入金额 ≤ 可用现金 - 最低现金保留 → 按优先级截断
- 单票不超风控上限 → 自动裁剪
- 标的必须可交易 → 交叉验证

#### 推理追踪（可选）

保存完整决策轨迹到 `results/trace_YYYYMMDD.json`，供后续审计：

```json
{
  "timestamp": "2026-05-25T09:30:00",
  "screening": { "total_stocks": 5123, "candidates": 20 },
  "analysis": { "600519": { ... } },
  "risk": { "600519": { "position_limit": 0.20, "max_shares": 500 } },
  "decision": [{"symbol": "600519", "symbol_name": "贵州茅台", "volume": 200}]
}
```

## 五、运行流程

### 5.1 交易日时间线

```
08:30  系统唤醒，检查数据源可用性
09:00  阶段一: 全市场海选 → Top 20 候选池
09:15  阶段二: 四维分析师并行分析
10:00  阶段三: 辩论式对抗 + 研究主管研判
10:30  阶段四: 风控计算 + 组合构建
10:45  输出赛道 JSON 格式决策
```

### 5.2 异常处理

| 场景 | 处理策略 |
|------|----------|
| 数据源全部不可用 | 输出空数组 `[]`，标记"数据不可用" |
| LLM 调用超时 | 降级为确定性规则引擎 (`fallback.py`) |
| 单只股票分析失败 | 跳过该股票，继续分析其他候选 |
| 组合构建失败 | 输出空数组 `[]` |
| 当日无符合条件标的 | 输出空数组 `[]` |

### 5.3 运行方式

```bash
# 正常模式 (需要 LLM API Key 和数据源)
python -m src.main

# 演示模式 (使用样本数据，无需网络和 API)
python -m src.main --demo

# 或在代码中调用
from src.graph.workflow import run_pipeline
state = run_pipeline(total_capital=500_000.0, available_cash=500_000.0)
```

## 六、策略可解释性

### 6.1 推理链路

每笔决策可从最终输出回溯到原始数据：

```
最终决策: 买入贵州茅台 200 股
  ↑
组合主管: "综合各维度看多信号，风控允许500股，建议保守配置200股"
  ↑
研究主管: "多头论点(业绩确定性+资金流入)强于空头(估值偏高+短期超买)"
  ↑
├─ 多头研究员: "Q1业绩超预期20%，北向资金连续5日净流入"
├─ 空头研究员: "PE处于历史70%分位，RSI接近超买"
├─ 技术面: bullish(0.75)  ├─ 基本面: bullish(0.82)
├─ 资金面: bullish(0.68)  └─ 消息面: bullish(0.70)
```

### 6.2 回溯审计

每交易日保留完整 JSON 轨迹文件，支持：
- 按日期查询任意决策的完整推理链
- 统计各分析师的预测准确率

## 八、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 数据源接口不稳定 | 数据缺失 | 三层降级 (Tushare→BaoStock→AKShare) + 缓存兜底 |
| LLM 调用成本过高 | 预算超支 | 海选筛选减少调用量 + quick/deep 分层 |
| 市场极端波动 | 大幅回撤 | 日内5%熔断 + 单票仓位上限 + 保留现金 |
| 过拟合历史数据 | 实盘表现差 | 辩论机制引入对抗观点 + 多分析师交叉验证 |
| 平台对接问题 | 无法提交 | 输出格式严格校验 + stdout 纯 JSON 输出 |
