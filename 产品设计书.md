# 「智投未来」A股日内投资智能体 — 产品设计书

## 一、项目概述

### 1.1 赛道目标

构建面向 A 股日内投资决策的智能体，每日输出可解释的投资建议，初始虚拟资金 50 万元，以赛期结束可用资金总额排名。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **博采众长** | 最大化复用 `TradingAgents-CN`、`ai-hedge-fund`、`daily_stock_analysis` 的成熟模块 |
| **可解释性** | 每笔决策附带完整推理链，数据→信号→决策全链路可审计 |
| **稳健性** | 多层数据源降级、异常熔断、空仓兜底，极端行情下不崩溃 |
| **模块化** | 数据层 / 分析层 / 决策层 / 输出层 松耦合，各层可独立替换 |

## 三、核心模块设计

### 3.1 数据层 — UnifiedDataInterface

**复用来源**: `TradingAgents-CN` 的 `dataflows/interface.py` + `daily_stock_analysis` 的 `DataFetcherManager`

**设计要点**:

```python
class UnifiedDataInterface:
    """统一数据接口，多源降级 + 缓存 + 数据质量标记"""

    # 数据源优先级 (Tushare → BaoStock → AKShare，按 Config 动态排序)
    # AKShare 已永久降级为兜底 (priority=999)，仅当 Tushare/BaoStock 均不可用时启用

    # ── 核心行情 ──
    def get_market_snapshot(self) -> list[MarketSnapshot]:
        """获取全市场快照: 涨跌幅、量比、换手率、成交额"""

    def get_daily_data(self, code: str, days: int = 60) -> list[StockDaily]:
        """获取个股日线: OHLCV + 涨跌幅 + 换手率 + 量比"""

    def get_realtime_quote(self, code: str) -> Optional[RealtimeQuote]:
        """获取实时行情: 最新价/涨跌幅/成交量/换手率"""

    def get_stock_name(self, code: str) -> str:
        """获取股票名称"""

    def get_stock_info(self, code: str) -> dict:
        """获取股票基本信息: 行业/市值/PE/PB"""

    # ── 资金面 ──
    def get_fund_flow(self, code: str, days: int = 5) -> list[FundFlow]:
        """获取资金流向: 主力/超大单/大单/中单/小单 净流入"""

    def get_northbound_flow(self, days: int = 5) -> list[NorthboundFlow]:
        """获取北向资金流向"""

    def get_northbound_stock(self, code: str, days: int = 10) -> list[dict]:
        """获取北向个股持仓变化"""

    def get_margin_summary(self) -> dict:
        """获取融资融券总览"""

    def get_margin_detail(self, code: str, days: int = 10) -> list[MarginData]:
        """获取个股融资融券明细"""

    def get_block_trades(self, days: int = 10) -> list[dict]:
        """获取大宗交易数据"""

    def get_dragon_tiger_stats(self, days: int = 10) -> list[dict]:

    def get_financial_indicators(self, code: str) -> list[FinancialIndicator]:

    def get_shareholder_count(self, code: str) -> list[ShareholderCount]:
        """获取股东人数变化 (筹码集中度)"""

    def get_unlock_shares(self, days_ahead: int = 30) -> list[UnlockShares]:
        """获取限售解禁信息"""

    # ── 消息面 ──
    def get_news(self, keyword: str, days: int = 3) -> list[dict]:
        """获取财经新闻"""

    def get_telegraph(self, limit: int = 30) -> list[dict]:
        """获取电报 (实时快讯)"""

    def get_research_reports(self, code: str, days: int = 30) -> list[dict]:
        """获取券商研报"""

    def get_institutional_visits(self, days: int = 30) -> list[InstitutionalVisit]:
        """获取机构调研信息"""

    def get_market_activity(self) -> list[MarketActivity]:
        """获取市场活跃度 (涨停/跌停/炸板)"""

    # ── ETF ──
    def get_etf_spot(self) -> list[ETFSpot]:
        """获取 ETF 实时行情"""

    def get_etf_daily(self, code: str, days: int = 60) -> list[StockDaily]:

    def get_data_quality(self, code: str, data_type: str) -> str:
        """查询数据质量标记: live / cached / fallback / stale"""
```

**降级策略** (从 daily_stock_analysis 复用):
- Tushare 不可用 → 尝试 BaoStock → 尝试 AKShare(兜底) → 返回缓存数据 → 标记数据质量等级
- 单只股票数据缺失不阻断整体流程
- 注意：财联社电报、机构调研、盘口异动 3 个 API 已确认废弃，直接返回空（不重试）

**数据质量标记**:
每条数据自动标记来源质量，全链路可追溯:

| 标记 | 含义 | 触发条件 |
|------|------|----------|
| `live` | 实时数据 | 主数据源 (Tushare) 成功返回 |
| `cached` | 缓存数据 | 在 TTL 内命中缓存，未重新请求 |
| `fallback` | 兜底数据 | 主源失败，降级到 BaoStock/AKShare |
| `stale` | 过期缓存 | 所有数据源均失败，返回过期缓存兜底 |

通过 `UnifiedDataInterface.get_data_quality(code, data_type)` 查询，数据质量写入 trace JSON 的 `data_quality` 字段。

### 3.2 分析层 — Multi-Agent Analysis Pipeline

#### 阶段一: 海选筛选 (Morning Screening)

**实现方式**: 自研 `ScreeningScorer` 类，融合 daily_stock_analysis、ai-hedge-fund 的多因子思想

**目标**: 从 5000+ A 股快速筛选 Top 20 候选，大幅降低 LLM 调用成本

**多因子打分模型**:

| 因子类别 | 因子名称 | 权重 | 说明 |
|----------|----------|------|------|
| 趋势 | 均线多头排列 | 12% | MA5>MA10>MA20>MA60 |
| 动量 | 5日涨幅 | 10% | 正向动量 |
| 量价 | 放量上涨 | 12% | 量比+涨幅连续评分 |
| 资金 | 主力净流入 | 15% | 大单净流入占比 |
| 北向 | 外资持仓变化 | 10% | 北向持股占比趋势 |
| 情绪 | 交易活跃度 | 8% | 换手率+连续阳线 (MVP替代新闻情感) |
| 质量 | PE+ROE+营收增速+经营现金流 | 10% | 多维度基本面过滤 |
| 风险 | 波动率适中 | 8% | 排除异常波动股 |
| 流动性 | 日均成交额 | 10% | >5000万，确保可交易 |
| 筹码 | 股东人数变化 | 5% | 筹码集中度指标 |

**筛选流程**:
```
5000+ 全市场
  → 剔除 ST/*ST/停牌/新股(上市<60天)
  → 剔除 日均成交额 < 5000万 (流动性过滤)
  → 10因子加权打分
  → Top 20 进入深度分析
```

#### 阶段二: 四维分析师并行分析

**复用来源**: `TradingAgents-CN` 的 `agents/analysts/` + `ai-hedge-fund` 并行 Agent 模式

**LLM 分层策略** (从 TradingAgents-CN 复用):

| 层级 | 模型 | 用途 | 调用频率 |
|------|------|------|----------|
| **quick** | DeepSeek-V3.2 / GPT-4o-mini | 分析师、研究员 | 高 (每只股票x4) |
| **deep** | DeepSeek-V4-Pro / Claude Opus | 研究主管、风控、组合主管 | 低 (全市场x1) |

四个分析师**并行运行** (从 ai-hedge-fund 复用)，各自使用 quick LLM:

```
                    ┌─ 技术面分析师 (Technical Analyst) ─┐
                    │  · 均线形态、MACD金叉死叉           │
                    │  · 布林带位置、RSI超买超卖          │
                    │  · 筹码分布、支撑压力位             │
候选池 (20只) ──────┼─ 基本面分析师 (Fundamentals Analyst) ─┼─→ 四份报告
                    │  · PE/PB分位数、ROE趋势             │
                    │  · 营收增速、利润质量               │
                    │  · 近期公告影响评估                 │
                    ├─ 资金面分析师 (Fund Flow Analyst) ───┤
                    │  · 主力资金动向、北向资金            │
                    │  · 融资融券变化                     │
                    │  · 大宗交易/龙虎榜                  │
                    └─ 消息面分析师 (News Sentiment Analyst)┘
                       · 财经新闻情感分析
                       · 行业/概念热点匹配
                       · 社交媒体热度
```

每个分析师遵循 **Tool-Calling 循环** (从 TradingAgents-CN 复用):
```
分析师LLM → 自主选择工具调用 → 获取数据 → 继续分析 → ... → 生成报告
```
最多 3 轮工具调用，防止无限循环。

#### 阶段三: 辩论式对抗

**复用来源**: `TradingAgents-CN` 的 Bull/Bear Researcher 辩论模式

```
多头研究员 (Bull Researcher):
  "基于技术面金叉+资金持续流入+业绩预增，
   建议买入 XXX，目标价 XX，止损 XX"
                    ↓
空头研究员 (Bear Researcher):        ← 基于多头论点逐条反驳
  "技术面虽有金叉但量能不足，
   资金流入可能为短期炒作，
   业绩预增已被市场price in..."
                    ↓
多头研究员 (Bull Researcher):        ← 回击空头论点
  ...
                    ↓
研究主管 (Research Manager, deep LLM):
  综合双方论点 → 给出最终研报
  {股票代码, 建议方向, 目标价, 置信度, 风险等级, 核心理由}
```

每只候选股票经过 1-3 轮辩论（可配置），研究主管用 deep LLM 做出最终研判。

### 3.3 决策层 — Risk-Adjusted Portfolio Construction

**复用来源**: `ai-hedge-fund` 的 `risk_manager.py` + `portfolio_manager.py`

#### 风控主管 (Risk Manager)

```python
class RiskManager:
    """风控规则 (确定性计算，不消耗 LLM Token)"""

    # 基本规则
    MAX_SINGLE_POSITION = 0.20    # 单票 ≤ 20% 总资金 (10万)
    MAX_INDUSTRY_EXPOSURE = 0.40  # 同行业 ≤ 40%
    MAX_DAILY_TURNOVER = 0.50     # 日换手率 ≤ 50% (避免过度交易)
    MAX_DRAWDOWN_DAILY = 0.05     # 日内回撤 5% 熔断
    MIN_CASH_RESERVE = 0.10       # 保留 ≥ 10% 现金

    def compute_position_limit(
        self,
        ticker: str,
        volatility: float,
        correlations: dict[str, float],
        current_portfolio: Portfolio
    ) -> PositionLimit:
        """
        波动率调整仓位上限 (从 ai-hedge-fund 复用)
        - 低波动 (σ < 2%): 1.25x 基准 → 最大 25%
        - 中波动 (2% ≤ σ < 4%): 1.00x 基准 → 最大 20%
        - 高波动 (σ ≥ 4%): 0.50x 基准 → 最大 10%

        + 相关性惩罚: 与已持仓股票高相关 (ρ > 0.7) → ×0.7
        """
```

```python
        """
        LLM 输入:
```

> **注**: 组合主管现已完整支持双向决策（买入+卖出）。LLM 根据持仓盈亏、持有天数、止损信号等主动判断卖出时机，`PortfolioTracker.apply_sells()` 在执行买入前先执行卖出以回收资金。`ResearchVerdict.direction` 的 `"sell"` 信号由组合主管从持仓组合层面独立判断，不直接依赖研报卖出信号。

## 五、运行流程

### 5.1 交易日时间线

```
08:30  系统唤醒，检查数据源可用性
09:00  阶段一: 全市场海选 → Top 20 候选池
09:15  阶段二: 四维分析师并行分析 (每只股票 ~30s)
10:00  阶段三: 辩论式对抗 + 研究主管研判
10:30  阶段四: 风控计算 + 组合构建
10:45  输出赛道 JSON 格式决策 → 提交至擂台平台
11:00  盘中监控启动 (可选: 熔断/止盈止损信号)
14:30  最终决策确认并输出
15:00  收盘后: 生成日报 + 更新记忆库 + 保存推理轨迹
```

### 5.2 异常处理

| 场景 | 处理策略 |
|------|----------|
| 数据源全部不可用 | 输出空数组 `[]`，标记"数据不可用" |
| LLM 调用超时 | 降级为确定性规则引擎 (`fallback.py`): 分析师→MA/MACD/RSI/PE 等规则指标、研究主管→多数投票、组合主管→保守 Top-N 配置 |
| 单只股票分析失败 | 跳过该股票，继续分析其他候选 |
| 组合构建失败 | 输出空数组 `[]`，保留现有持仓 |
| 当日无符合条件标的 | 输出空数组 `[]` |

### 5.3 命令行参数


```
├─ 技术面: 均线多头排列，MACD金叉，成交量放大 → bullish(0.75)
├─ 基本面: ROE 28%，营收增速 18%，PE 32 (合理偏高) → bullish(0.82)
├─ 资金面: 主力净流入 2.3亿，融资余额上升 → bullish(0.68)
```

### 第四阶段 (前沿) — 进行中

#### Transformer 时序编码器 ✅ 已完成

轻量级 Transformer 编码器 (`src/transformer/`)，从原始 K 线序列中自动学习股票的时序表示，替代/增强手工特征。

**架构规格**:

| 参数 | 值 | 说明 |
|------|-----|------|
| d_model | 32 | 嵌入维度 |
| n_heads | 4 | 多头注意力头数 |
| d_ff | 64 | FFN 隐藏层 |
| n_layers | 2 | 编码器层数 |
| max_seq_len | 60 | 最大序列长度 |
| n_features | 10 | 输入特征 (open/high/low/close/volume/amount/pct_chg/turnover/macd_bar/rsi_14) |

**数据流**: `StockDaily 序列 (30天×10特征) → FeatureEmbedding + PositionalEncoding → TransformerEncoder ×2 → Mean Pooling → Output Head → 预测收益率 → 映射为 [5,95] 评分`

**集成方式**:
- 主集成：作为第 11 个因子增强 `ScreeningScorer`，`composite = 手工×0.7 + transformer×0.3`（权重可配）
- 辅集成：用 Transformer 编码替代 `compute_features()` 的 7 维手工特征提供给 RL

**使用方式**:
```bash
python -m src.main --transformer-train --transformer-epochs 100  # 训练
export TRANSFORMER_ENABLED=true                                    # 启用
export TRANSFORMER_MODEL_PATH=results/transformer_model.json       # 模型路径
python -m src.main --demo                                          # 推理
```

**设计特点**: 纯 Python 实现(无 PyTorch/numpy 依赖)，约 18K 参数，JSON 序列化，过拟合测试 loss < 0.01

## 九、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 数据源接口不稳定 | 数据缺失 | 三层降级 (Tushare→BaoStock→AKShare) + 缓存兜底 (dataclass 序列化) |
| LLM 调用成本过高 | 预算超支 | 海选筛选减少 deep LLM 调用 + quick/deep 分层 |
| 市场极端波动 | 大幅回撤 | 日内5%熔断 + 单票仓位上限 + 保留现金 |
| 过拟合历史数据 | 实盘表现差 | 辩论机制引入对抗观点 + 多家分析师交叉验证 |
| 擂台平台对接问题 | 无法提交 | 输出格式严格校验 + 模拟接口预测试 |
